平台说明
================================================================================
GCC工程，由Eclipse IDE外挂GNU-ARM Toolchain，再结合pyOCD GDB Server实现工程的编译、
链接和调试。在用Eclipse导入工程后，请将xxxx_PyOCDDebug中pyocd-gdbserver和SVD文件
设置为正确的路径；请将xxxx_PyOCDDownload中pyocd设置为正确的路径。注意，这些路径不
能包含非英文字符。


功能描述
================================================================================
本例程用ADTRG_A（P14）作为序列A的触发源（ADTRG_A由高到低的跳变，并保持低电平6个HCLK周期
以上）；用外部中断0事件EVT_PORT_EIRQ0作为序列B的触发源（拉低引脚P17可产生该事件）。例程
展示了用外部引脚和内部事件作为触发源的配置方法。

================================================================================
使用步骤
================================================================================
1）USB转串口工具连接至USART3，并与电脑相连；用MicroUSB将开发板与电脑相连；
2）打开工程，重新编译，下载至开发板并运行；
3）拉低P14，触发序列A动作，可看到串口调试助手显示序列A被触发的信息；
4）拉低P17，可看到串口调试助手显示序列B被触发的信息；
5）也可屏蔽串口打印，通过断点调试的方式查看程序运行情况。


================================================================================
注意
================================================================================
ADC有三个外部触发引脚（P14，P30和P60），它们的功能是一样的，三个引脚与序列A和序列B无
对应关系，其中一个序列选择了外部引脚作为触发源，另外一个序列就不可以再选择外部引脚作为
触发源。如果选择内部事件作为触发源，两个序列不可设置相同的事件作为触发源。
